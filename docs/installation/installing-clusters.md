# Installing clusters

Once the [operator is installed](installation-management.md), you define your infrastructure by creating `Cluster` resources in the `superphenix-system` namespace. These resources tell the operator where and how to deploy the Superphenix stack on each availability zone.

## In this section

- **[Configure a cluster](configure-cluster.md)** — topology, geography, connection mode, and system configuration for a `Cluster` resource.
- **[Connecting clusters](connecting-clusters.md)** — link virtualization clusters to storage backends in decoupled deployments.
